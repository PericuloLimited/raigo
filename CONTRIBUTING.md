# Contributing to RAIGO

Thank you for your interest in contributing to the RAIGO open standard. This project is maintained by [Periculo Security](https://periculo.com) and is open to contributions from the community.

## How to Contribute

### 1. Improving the Specification
If you have suggestions for new fields, action types, or schema improvements, please open an issue with the label `spec-proposal` and describe the use case.

### 2. Adding Example Files
The `/examples` directory is the most impactful place to contribute. If you have a `.raigo` file for a specific industry or compliance framework (e.g., SOC 2, ISO 27001, GDPR, FedRAMP), please submit a pull request.

### 3. Compiler Targets
If you want to add a new compilation target (e.g., a new AI tool or workflow platform), open an issue with the label `new-target` and we will work with you to define the output format.

## Code of Conduct
Be excellent to each other. This is a security-focused project and we take responsible disclosure seriously.
